# tslib Explanation

## What is tslib?

`tslib` is TypeScript's runtime library that contains helper functions generated by the TypeScript compiler when compiling code.

## Why is tslib needed?

When the TypeScript compiler compiles TypeScript code to JavaScript, certain language features require additional runtime support code. For example:

1. **Decorators**: Require helper functions like `__decorate`, `__metadata`, etc.
2. **Class inheritance**: Requires `__extends` helper function
3. **Object properties**: Require `__assign`, `__spread` helper functions
4. **Async generators**: Require `__asyncGenerator`, `__await` helper functions
5. **Optional chaining and nullish coalescing**: Require helper functions in certain compilation targets

## Usage in jsMind V2 Project

In the jsMind V2 project, we use the `@rollup/plugin-typescript` plugin to compile TypeScript code. This plugin will:

1. Detect TypeScript features used in the code
2. Attempt to import helper functions from `tslib` if needed
3. Fail with an error if `tslib` is not installed

## Installation

```bash
npm install --save-dev tslib
```

## Configuration Options

In the TypeScript configuration (`tsconfig.json`), you can control whether to use tslib through the `importHelpers` option:

- `"importHelpers": true` - Import helper functions from tslib (recommended, reduces bundle size)
- `"importHelpers": false` - Inline helper functions in each file (not recommended, increases bundle size)

## Summary

`tslib` is a required development dependency that provides runtime helper functions needed for TypeScript compilation. Although our code may not directly use these functions, the TypeScript compiler and Rollup plugin need them to correctly compile and bundle the code.
